# ADR-0006: 盤面の正規化キーで対称性を吸収する

- Status: Accepted
- Date: 2026-05-04

## 背景

立体4目並べでは、ゲームの本質的な進行は同じでも、盤面を回転・鏡映させると異なる3進キーが生成されてしまう。

重力ありのルールにおいて、盤面が持つ幾何学的な対称性は以下の通り:

- **z軸中心の回転**: 90度単位で4通り（0°, 90°, 180°, 270°）
- **x-y平面内の鏡映**: x座標またはy座標の反転

z軸（上下）反転は重力方向を逆にしてしまうため、ゲーム的に異なる局面になり、対称性として扱えない。

回転4通り × 鏡映2通りの組み合わせから、重複を除いて**最大8通り**の対称な盤面表現が存在する。

現在の `MemoTable` は `board_key_base3()` をそのまま使うため、対称な8つの局面を別物として記録し、探索量とメモリ消費が8倍になる。

## 決定

- `Board` に `normalized_key()` メソッドを追加する。
- 以下の8通りの変換を適用した盤面キーを生成し、そのうち**最小値**を正規化キーとして返す:
  1. 元の盤面
  2. z軸中心に90度回転
  3. z軸中心に180度回転
  4. z軸中心に270度回転
  5. x座標反転後、元の向き
  6. x座標反転後、z軸中心に90度回転
  7. x座標反転後、z軸中心に180度回転
  8. x座標反転後、z軸中心に270度回転
- `MemoTable` の `insert` と `get` は `normalized_key()` を使うように変更する。
- 既存の `board_key_base3()` はそのまま残し、正規化しないキーが必要な場合にも使えるようにする。
- 正規化キーは探索結果・評価結果を引くためのキーとして使う。実際の対局中の現在盤面や合法手座標は、正規化前の `GameState` から扱う。

## 理由

- 最小値を選ぶことで、8通りの対称形が必ず同じキーに収束する。
- z軸反転を含めないことで、重力ルールと矛盾しない対称性だけを扱う。
- x座標反転のみを鏡映に使うのは、y座標反転やxy両座標反転が回転と鏡映の組み合わせで表現できるため。
- 変換を全列挙してから最小値を取るのは、コードが読みやすく、対称性の網羅性が確認しやすいため。

## 影響

- `MemoTable` を使う探索の効率が最大8倍向上する。
- 初期状態の探索や、大規模な状態数カウントが現実的になる。
- 正規化の計算コストは追加されるが、探索の重複削減効果の方が遥かに大きい。
- `Board` の役割が「盤面表現」から「盤面表現と対称性の正規化」へ拡張される。
- 将来、ビット盤など別のキー表現を導入する場合も、正規化の考え方は同じ枠組みで扱える。
- 正規化キーだけでは、元の盤面からどの回転・鏡映で代表キーに到達したかは分からない。そのため、正規化キーだけを保存しても「元の盤面でどの手を指すべきか」は直接復元できない。
- 対戦AIで最適手を選ぶ場合は、正規化前の現在盤面から合法手を列挙し、各合法手を打った後の局面を `normalized_key()` で評価表に問い合わせる。最適手そのものを正規化キーに紐づけて保存したい場合は、回転・鏡映の変換情報も別途必要になる。
